ssh学习（1）

发表于 2015-12-03   |   分类于 网络

ssh介绍

ssh为secure shell的缩写，是建立在应用层和传输层基础上，目前较可靠，专为远程登录会话和其他网络服务提供安全性的协议。使用ssh可以有效防止远程管理的信息泄露问题。

功能（纯粘贴，然并卵）

通过使用SSH，你可以把所有传输的数据进行加密。也能够防止DNS欺骗和IP欺骗。使用SSH，还有一个额外的好处就是传输的数据是经过压缩的，所以可以加快传输的速度。SSH有很多功能，它既可以代替Telnet，又可以为FTP、PoP、甚至为PPP提供一个安全的”通道”。

ssh的验证

从客户端的角度看，有两种级别的验证

第一种级别

知道账号和口令，就可以登录到远程主机，所有数据都是加密的，但不能确保连接到的主机就是你想连接的服务器，可能会有“假”服务器在冒充。

第二种级别

密钥机制，ssh的密钥是自己创造一对密钥，并把公用密钥放在需要访问的服务器上。如果需要连接ssh服务器，客户端软件就会向服务器发送公用密钥的请求，服务器收到请求后，先在该服务器上你的主目录下寻找你的公用密匙，然后把它和你发送过来的公用密匙进行比较。如果两个密匙一致，服务器就用公用密匙加密“质询”（challenge）并把它发送给客户端软件。客户端软件收到“质询”之后就可以用你的私人密匙解密再把它发送给服务器。

结构

SSH是由客户端和服务端的软件组成的

服务端

服务端是一个守护进程？，他在后台运行并响应来自客户端的连接请求。服务端一般是sshd进程，提供了对远程连接的处理，一般包括公共密钥认证、密钥交换、对称密钥加密和非安全连接。

客户端

客户端包含ssh程序以及像scp（远程拷贝）、slogin（远程登陆）、sftp（安全文件传输）等其他的应用程序。

工作机制

他们的工作机制大致是本地的客户端发送一个连接请求到远程的服务端，服务端检查申请的包和IP地址再发送密钥给SSH的客户端，本地再将密钥发回给服务端，自此连接建立。
一旦建立一个安全传输层连接，客户机就发送一个服务请求。当用户认证完成之后，会发送第二个服务请求。这样就允许新定义的协议可以与上述协议共存。

小结

多为粘贴和照抄，主要是为了了解

mac环境变量配置

发表于 2015-12-02   |   分类于 操作系统 ， mac

由于mac下使用的shell是zsh，所以在配置好某个环境变量，重启item2就会发现刚配置好的环境变量失效。
zsh在启动时自动加载的是.zshrc文件，所以可以通过修改该文件来配置永久的环境变量
命令行下输入vim ~/.zshrc，找到“# User configuration”部分。
在export PATH= xxx下配置需要的环境变量。
注意每个环境变量是用“：”符分割的。

Scala 构造器

主构造器

主构造器的特点：

1.主构造器直接跟在类名后面（Person），主构造器中的参数被默认编译为类的字段
2.主构造器执行时会执行类中的所有不包含在方法体（def）中的语句
3.主构造器参数必须用val或者var声明，否则编译器默认编译为private[this]

附属构造器

1.附属构造器用this声明
2.附属构造器中必须调用主构造器或其他附属构造器

Linux常用命令（文件和目录）

发表于 2015-08-12   |   分类于 Linux

Linux文件目录

ls命令常用参数

参数名称 描述
-a (—all) 输出包括以“.”开头的隐藏文件
-c 按最后一次修改时间排序
-F 区别目录和文件
-i (—inode) 显示每个文件的索引值
-l 按长列表输出格式显示
-L 对于链接文件，显示原文件信息
-o 在长列表格式下不要显示组名
-r (—reverse) 输出文件和目录时反转排序的顺序
-R (—recursive) 递归的列出子目录内容
-s (—size) 输出每个文件的块大小
-S (—sort) 按文件大小排序输出
-t (—time) 按文件的修改时间排序输出
-v (—version) 按文件版本排序输出

如果需要，可以一次使用多个参数。多个双破折线参数必须分开输入，而多个单破折线可以组成一个字符串跟在一个单破折线后面。常见的组合是，用-a参数去列出所有文件，用-i参数列出每个文件的索引点，用-l生成一个长列表，再用-s参数列出所有文件的块大小。组合起来就是-sails参数。

处理文件

创建文件命令：touch

复制文件命令：cp，cp命令需要两个参数，源对象和目标对象：
cp source destination

重命名文件：mv file1 file2

删除文件：rm file
